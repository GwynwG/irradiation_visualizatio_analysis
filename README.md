# 辐照监测可视化与智能化分析系统

本系统用于对来自不同房间、不同设备的辐照监测数据进行可视化表达、质量校验、异常识别、趋势预测、风险排序和 Excel 分析报告导出。

## 运行方式

```powershell
python -m pip install -r requirements.txt
streamlit run app.py
streamlit run report_app.py
python -m pytest -q
```

## 数据范围

- 房间编号：`R01` 至 `R20`
- 设备编号：每个房间 `D01` 至 `D10`
- 设备 ID 示例：`R01-D01`、`R20-D10`
- 总设备数：20 个房间 x 10 台设备 = 200 个设备

## Excel 输入格式

系统读取 `.xlsx` 工作簿。监测数据通常位于 `监测数据` 工作表，也可以在同一工作簿内包含 `设备阈值配置` 等辅助工作表。导入器会自动扫描包含必需列的工作表；当同一文件存在多个候选工作表时，可在界面中手动选择。

必需列：

| 列名 | 说明 |
| --- | --- |
| 监测时间 | 日期或日期时间 |
| 房间ID | 如 `R01` |
| 设备ID | 如 `R01-D01` |
| 监测类型 | 如 `γ剂量率`、`中子剂量率` |
| 监测值 | 数值 |
| 单位 | 如 `μSv/h` |
| 预警值 | 人工干预预警阈值 |
| 控制标准 | 法定控制标准 |

可选列：

| 列名 | 说明 |
| --- | --- |
| 房间名称 | 房间显示名称 |
| 设备名称 | 设备显示名称 |
| 数据来源 | 记录来源 |
| 备注 | 补充说明 |

阈值边界：

- 监测值 `>= 控制标准`：事故级。
- 监测值 `>= 预警值` 且 `< 控制标准`：预警。
- 监测值 `< 预警值`：正常。
- 非正数阈值不会触发对应级别判定。

## 四阶段分析工作台

1. 数据导入：上传一个或多个 `.xlsx` 文件，选择候选工作表，查看导入摘要、质量问题和有效记录预览。
2. 空间总览：按时间点或时间范围构建快照，使用 V5 房间布局图查看 20 个房间和 200 台设备的状态。
3. 趋势分析：查看设备序列曲线、预警/控制标准变化、房间异常设备数量趋势和异常事件表。
4. 智能研判：查看设备风险、房间风险、快速增长、接近预警和预测结果。

## 独立报告生成器

报告生成不再集成在主工作台中，需单独启动：

```powershell
streamlit run report_app.py
```

报告生成器独立完成工作簿上传、候选工作表识别、数据校验、预测周期选择、多工作表分析报告下载、模板下载和模拟数据下载。

## V5 设施布局

布局为 4 排房间，每排 5 个房间，并在第 1/2 排和第 3/4 排之间设置主过道：

- 第 1 排：`R01` - `R05`
- 主过道 A
- 第 2 排：`R06` - `R10`
- 第 3 排：`R11` - `R15`
- 主过道 B
- 第 4 排：`R16` - `R20`

每个房间包含 10 台设备，并采用 A/B/C 三类设备摆放方式。入口统一标注为 `入口`，设备状态颜色与报告一致：

- 无数据：灰色
- 正常：绿色
- 预警：橙色
- 事故级：红色

## 分析与预测

系统提供以下智能化分析结果：

- 数据质量评分：综合阻断行、冲突键、完全重复行和提示类问题，给出质量分、等级、扣分原因和问题分布。
- 异常事件：连续预警/事故级记录形成事件，正常记录关闭事件。
- 接近预警：默认识别达到预警值 80% 以上但尚未预警的记录。
- 快速增长：按设备、监测类型和单位识别近期增量显著高于历史波动的序列。
- 智能预警：统一汇总当前超阈、接近预警、快速增长、持续异常事件和预测超阈信号，并给出证据与建议动作。
- 风险排序：综合严重程度、超阈幅度、持续时间、趋势和复发情况。
- 简单预测：提供最近值、移动平均、线性趋势和指数平滑候选方法，并给出方法、置信度和说明。

预测仅用于趋势研判，不替代现场复核、仪器校准或安全处置决策。

## 导出内容

多工作表分析报告包含：

1. `分析摘要`
2. `智能预警`
3. `异常事件`
4. `设备风险排名`
5. `房间风险排名`
6. `趋势预测`
7. `清洗后的监测数据`
8. `数据质量问题`

报告会保留源文件、源工作表和源行号，便于追溯。
`分析摘要` 会写入质量分和质量等级，`数据质量问题` 工作表会先展示质量评分概览，再列出明细问题。
