# 辐照监测系统升级路线

## 目标边界

主分析工作台聚焦数据导入、空间态势、趋势分析和智能研判。报告生成作为独立应用运行，通过 `report_app.py` 提供报告、模板和模拟数据能力，不作为主工作台的 Tab 或子页面。

## 一期：已落地的结构调整

- 主工作台从五阶段调整为四阶段：数据导入、空间总览、趋势分析、智能研判。
- 新增独立报告入口：`streamlit run report_app.py`。
- 抽出 `report_service.py`，让报告 UI 调用服务层，而不是把报告编排逻辑绑在页面函数内。
- 独立报告器支持预测周期选择，生成报告时可选择下一条记录、未来 1 天、未来 7 天或未来 30 天。
- 新增统一智能预警引擎：覆盖当前超阈、接近预警、快速增长、持续异常事件和预测超阈信号。
- 智能研判页新增预警总览、预警分布图和证据表。
- Excel 分析报告新增 `智能预警` 工作表，记录触发规则、证据和建议动作。
- 新增数据质量评分：按阻断行、冲突键、完全重复行和提示类问题形成质量分、等级、扣分原因和问题分布。
- 导入页新增质量评分概览和问题代码分布图。
- Excel 报告摘要和 `数据质量问题` 工作表新增质量评分概览。

## 二期：UI 升级

- 建立统一的页面骨架：主状态区、过滤区、图表区和证据表区固定位置，减少跨页面认知成本。
- 增强空间总览的操作反馈：设备悬停信息、选中设备高亮、异常房间快速定位。
- 将关键指标收敛为稳定的信息密度：导入质量、当前事故级设备、预警设备、预测风险设备优先展示。
- 把视觉体系从单一蓝色调整为低饱和中性色、辐照监测绿、风险琥珀和事故红，避免模板化仪表盘观感。

## 三期：算法升级

- 风险评分参数配置化：严重度、超阈幅度、持续时间、增长趋势和复发权重可配置。
- 预测方法增加模型对比记录：保存每个候选方法的误差、选择原因和样本覆盖范围。
- 增加稳健异常检测：在现有阈值判定之外，引入滑动分位数、MAD 或 EWMA 控制图，识别缓慢漂移。
- 增强数据质量审计：继续补充时间间隔异常、序列缺口和设备覆盖异常。

## 四期：功能模块补充

- 阈值配置模块：支持从 Excel 导入，也支持在界面中按房间、设备、监测类型维护。
- 事件处置模块：为异常事件添加确认、备注、处置状态和复核时间。
- 历史批次管理：保存多次导入的批次摘要，支持对比两个批次的风险变化。
- 权限与审计预留：后续接入登录后，可记录导入、阈值修改、报告生成等操作。

## 五期：结果可视化升级

- 空间态势图增加风险热力层和趋势方向标记。
- 趋势页增加预测区间、阈值变更标记和异常事件背景带。
- 智能研判页增加风险构成瀑布图，展示每台设备为什么高风险。
- 报告生成器输出可选图表工作表，保留趋势图、空间状态摘要和风险分布。

## 六期：智能预警与分析升级

- 预警策略分级：接近预警、快速增长、持续预警、事故级和预测超阈已形成独立规则。
- 预警解释标准化：每条预警已给出触发规则、证据、阈值、当前或预测值和建议复核动作。
- 趋势预警前置：对尚未超阈但预测窗口内可能超阈的序列给出提前关注。
- 形成房间级联动分析：同房间多设备同步上升时提升房间风险等级。

## 验证要求

- 单元测试覆盖导入、质量评分、阈值判定、预测、风险排序、报告生成和模拟数据。
- Streamlit smoke 测试同时覆盖 `app.py` 和 `report_app.py`。
- UI 相关改动需至少检查桌面宽屏和窄屏布局，确保按钮、表格和图表不互相遮挡。
