# 辐照监测可视化与智能化分析系统
## 使用说明书

文档版本：V2.1
适用软件：`app.py`（Streamlit 四阶段分析工作台）、`report_app.py`（独立报告生成器）
编制日期：2026-06-17

---

## 一、系统用途

本系统用于接入来自不同房间、不同设备的辐照监测 Excel 数据，完成数据质量校验、空间可视化、阈值状态判定、趋势分析和智能研判。工作表分析报告由独立报告生成器完成。

系统默认管理 20 个房间，每个房间 10 台设备，共 200 个设备。设备 ID 使用结构化编号，例如 `R01-D01` 表示 1 号房间的 1 号设备，`R20-D10` 表示 20 号房间的 10 号设备。

## 二、运行方式

### 2.1 安装依赖

```powershell
python -m pip install -r requirements.txt
```

### 2.2 启动主分析工作台

```powershell
streamlit run app.py
```

### 2.3 启动独立报告生成器

```powershell
streamlit run report_app.py
```

### 2.4 运行自动化测试

```powershell
python -m pytest -q
```

## 三、Excel 数据要求

系统读取 `.xlsx` 文件。监测数据通常位于 `监测数据` 工作表，也可以和 `设备阈值配置` 等辅助内容放在同一个工作簿中。导入时系统会扫描包含必需列的工作表；如果一个文件内存在多个候选工作表，界面会要求用户选择。

### 3.1 必需列

| 列名 | 说明 |
| --- | --- |
| 监测时间 | 日期或日期时间 |
| 房间ID | `R01` 至 `R20` |
| 设备ID | `R01-D01` 至 `R20-D10` |
| 监测类型 | 例如 `γ剂量率`、`中子剂量率` |
| 监测值 | 设备本次监测数值 |
| 单位 | 例如 `μSv/h` |
| 预警值 | 需要人为干预的预警阈值 |
| 控制标准 | 更严重的法定控制标准 |

### 3.2 可选列

| 列名 | 说明 |
| --- | --- |
| 房间名称 | 房间展示名称 |
| 设备名称 | 设备展示名称 |
| 数据来源 | 数据来源或批次 |
| 备注 | 补充说明 |

### 3.3 阈值判定规则

1. 监测值 `>= 控制标准`：判定为事故级。
2. 监测值 `>= 预警值` 且 `< 控制标准`：判定为预警。
3. 监测值 `< 预警值`：判定为正常。
4. 控制标准应高于或等于预警值；不符合该关系的记录会被阻止导入。
5. 每条监测记录可以重复填写预警值和控制标准，系统会保留阈值变化并在质量问题中提示。

## 四、四阶段分析工作台

### 4.1 数据导入

1. 上传一个或多个 `.xlsx` 工作簿。
2. 当系统发现多个候选工作表时，选择本次需要导入的工作表。
3. 点击导入后查看摘要，包括文件数、原始行数、有效行数、阻止行数、重复行和冲突键数量。
4. 查看质量分、质量等级、扣分原因和问题代码分布。
5. 在质量问题表中复核无效 ID、非数值、阈值关系错误、单位变化和阈值变化等问题。
6. 在有效记录预览中确认房间、设备、监测类型、监测值和源行号。

### 4.2 空间总览

空间总览使用经确认的 V5 设施布局：

1. 20 个房间分为 4 排，每排 5 个房间。
2. 第 1/2 排之间设置 `主过道 A`，第 3/4 排之间设置 `主过道 B`。
3. 每个房间有 10 台设备，并采用 A/B/C 三类排布方式。
4. 房间入口统一标注为 `入口`，入口朝向最近主过道或操作通道。

设备颜色表示当前状态：

| 颜色 | 状态 |
| --- | --- |
| 灰色 | 无数据 |
| 绿色 | 正常 |
| 橙色 | 预警 |
| 红色 | 事故级 |

可使用时间点或时间范围生成快照，并通过房间、设备、监测类型和状态筛选视图。点击设备后，右侧会展示该设备的最新值、阈值、记录时间、异常事件和预测摘要。

### 4.3 趋势分析

趋势分析用于查看长期监测变化：

1. 选择设备、监测类型和单位。
2. 查看监测值曲线，以及随时间变化的预警值和控制标准线。
3. 查看房间维度的异常设备数量趋势。
4. 查看异常事件表，包括事件起止时间、最高状态、峰值和持续信息。

### 4.4 智能研判

智能研判提供辅助分析结果：

1. 设备风险排名：综合严重程度、超阈幅度、持续时间、趋势和复发情况。
2. 房间风险排名：结合房间内最高设备风险、异常设备占比和持续信息。
3. 接近预警：默认识别达到预警值 80% 以上但尚未预警的记录。
4. 快速增长：识别近期增长明显高于历史波动的设备序列。
5. 智能预警：统一汇总当前超阈、接近预警、快速增长、持续异常事件和预测超阈信号，并展示触发证据、规则评分和建议动作。
6. 趋势预测：支持下一条记录、未来 1 天、未来 7 天和未来 30 天等预测窗口。

预测结果会显示方法、样本数量、置信度和解释。智能预警中的建议动作仅用于辅助排查，预测和预警均不替代现场复核、仪器校准、安全处置或事故调查。

## 五、独立报告生成器

报告生成器通过 `streamlit run report_app.py` 单独启动，不集成在主分析工作台内。

报告生成器提供三类能力：

1. 上传一个或多个监测工作簿，选择候选工作表，生成多工作表分析报告。
2. 下载空白监测数据模板和预填 200 台设备、默认监测类型的模板。
3. 生成可复现模拟数据，用于演示、测试和培训。

## 六、分析报告内容

工作表分析报告为 `.xlsx` 文件，包含：

| 工作表 | 内容 |
| --- | --- |
| 分析摘要 | 导入摘要、数据质量等级、快照状态、系统预测概览 |
| 智能预警 | 预警级别、评分、触发规则、证据和建议动作 |
| 异常事件 | 连续预警或事故级事件 |
| 设备风险排名 | 设备级风险分数、证据和原因 |
| 房间风险排名 | 房间级风险分数、异常设备情况 |
| 趋势预测 | 各设备序列预测值、状态、方法和置信度 |
| 清洗后的监测数据 | 去重、冲突处理后的有效记录 |
| 数据质量问题 | 质量评分概览、导入校验、冲突、单位变化和阈值变化 |

报告保留源文件、源工作表和源行号，便于追溯原始数据。

## 七、常见问题

### 7.1 为什么记录被阻止导入？

常见原因包括：房间 ID 不在 `R01-R20` 范围内、设备 ID 与房间不匹配、数值为空或非数字、预警值高于控制标准、监测时间无法识别。

### 7.2 同一设备可以有多个监测值吗？

可以。同一设备可按不同 `监测类型` 和 `单位` 形成不同序列，例如 `γ剂量率` 和 `中子剂量率`。设备状态会取这些序列中的最严重状态。

### 7.3 阈值后续能手动调整吗？

当前版本以 Excel 记录中重复填写的预警值和控制标准为准，并会提示阈值变化。界面化阈值配置和手动调整可作为后续功能扩展。

### 7.4 数据冲突如何处理？

若同一时间、同一房间、同一设备、同一监测类型存在多条不同记录，系统会保留导入顺序靠后的记录，并在质量问题中记录冲突证据。

---

## 八、注意事项

1. 请优先使用系统生成的模板填写数据，避免列名不一致。
2. 控制标准代表更严重的法定控制边界，应高于或等于预警值。
3. 监测数据跨很长时间时，建议保留完整日期时间，以提高趋势和预测结果的可解释性。
4. 预测和风险排名是辅助研判工具，不应作为唯一处置依据。
5. 导出报告后，建议结合源工作簿和现场记录进行复核。
